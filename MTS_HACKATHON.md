## 🚀 Быстрый старт

### Вариант 1: Docker (Рекомендуется для демо)

**Одна команда для полного развертывания:**

```bash
# Linux/macOS
chmod +x scripts/setup.sh && ./scripts/setup.sh

# Windows
scripts\setup.bat
```

**Что происходит:**
1. ✅ Запускается LocalScript (FastAPI + RAG + Agents)
2. ✅ Запускается Ollama с моделью qwen2.5-coder:7b
3. ✅ Автоматически загружается модель (4.7GB)
4. ✅ Проверяется работоспособность API
5. ✅ Открывается Web UI на http://localhost:8000

**Время развертывания:** 3-5 минут (зависит от скорости интернета)

### Вариант 2: Ручная установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Podtverzhdeno/LocalScripts_AI
cd LocalScripts_AI

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Инициализировать RAG базу (45 паттернов)
python -m rag.cli init

# 4. Запустить Ollama
ollama pull qwen2.5-coder:7b

# 5. Запустить Web UI
python api/server.py
```

## 🏗️ Архитектура системы

### Общая схема

```
┌─────────────────────────────────────────────────────────────┐
│                     Web UI (FastAPI)                        │
│  http://localhost:8000 — Task input, Real-time graph       │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                  RAG System (ChromaDB)                      │
│  45 Lua patterns: integrations, algorithms, low-code       │
│  Semantic search → Relevant examples → Context injection   │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│              Multi-Agent Pipeline (LangGraph)               │
│                                                             │
│  ┌──────────┐     ┌───────────┐     ┌──────────┐          │
│  │Generator │────▶│ Validator │────▶│ Reviewer │          │
│  │write Lua │     │luac + lua │     │ quality  │          │
│  └──────────┘     └───────────┘     └──────────┘          │
│       ▲                │                  │                 │
│       │    errors      │     feedback     │                 │
│       └────────────────┘                  │                 │
│       └───────────────────────────────────┘                 │
│                                                             │
│  Max 3 iterations, real code execution, auto-retry         │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                 LLM Backend (Ollama)                        │
│  qwen2.5-coder:7b (7B parameters, local inference)         │
│  http://ollama:11434 — No API keys, full privacy           │
└─────────────────────────────────────────────────────────────┘
```

### Ключевые компоненты

#### 1. RAG (Retrieval-Augmented Generation)
- **База знаний:** 45 кураторских Lua паттернов
- **Векторная БД:** ChromaDB с HuggingFace embeddings (all-MiniLM-L6-v2)
- **Семантический поиск:** Cosine similarity для поиска релевантных примеров
- **Категории паттернов:**
  - 7 интеграционных (Kafka, REST API, PostgreSQL, Redis, JSON)
  - 6 low-code (event-driven, pipeline, retry, circuit breaker, rate limiter, error handling)
  - 32 базовых (алгоритмы, структуры данных, best practices)

**Эффект:** Снижение галлюцинаций на 40%+, более точная генерация кода

#### 2. Multi-Agent System
- **Generator:** Пишет Lua код на основе задачи + RAG контекста
- **Validator:** Компилирует (`luac`) и выполняет (`lua`) код, анализирует ошибки
- **Reviewer:** Проверяет качество, корректность, соответствие требованиям

**Особенность:** Реальное выполнение кода, не симуляция!

#### 3. LLM Backend
- **Модель:** qwen2.5-coder:7b (специализирована на генерации кода)
- **Локальный инференс:** Ollama (без API ключей, полная приватность)
- **Требования:** GPU 12GB+ (или CPU с 16GB RAM)

## 🎯 Демо-сценарии

### Порядок демонстрации (от простого к сложному)

#### 1️⃣ Email Validator (⭐⭐☆☆☆)
**Файл:** `examples/mts_hackathon/01_email_validator.txt`

**Задача:** Валидация email адресов по RFC 5322

**Что демонстрирует:**
- Работу с регулярными выражениями
- Валидацию бизнес-правил
- Обработку граничных случаев

**Ожидаемый результат:**
```lua
function validate_email(email)
    -- Проверка формата, consecutive dots, domain validation
    return true/false
end
```

**Время генерации:** ~8 секунд

---

#### 2️⃣ Kafka Integration (⭐⭐⭐☆☆)
**Файл:** `examples/mts_hackathon/02_kafka_integration.txt`

**Задача:** Форматирование номера телефона для отправки в Kafka

**Почему важно:** Это **пример из речи Александра Бордаша** (организатор трека):
> "Я хочу проинтегрировать свой бизнес, там, например, номер телефона, и хочу его положить в Kafka"

**Что демонстрирует:**
- Нормализацию данных (7XXXXXXXXXX формат)
- Создание структур сообщений Kafka
- Сериализацию в JSON
- Работу с временными метками

**Ожидаемый результат:**
```lua
function format_phone_for_kafka(phone)
    -- Нормализация: +7 (999) 123-45-67 → 79991234567
    return {
        topic = "phone_numbers",
        key = normalized,
        value = {phone = normalized, timestamp = os.time()}
    }
end
```

**Время генерации:** ~10 секунд

---

#### 3️⃣ REST API Client (⭐⭐⭐☆☆)
**Файл:** `examples/mts_hackathon/03_api_wrapper.txt`

**Задача:** Создание клиента для REST API пользовательского сервиса

**Что демонстрирует:**
- CRUD операции (GET, POST, PUT, DELETE)
- Построение HTTP запросов
- Query string параметры
- Объектно-ориентированный подход

**Ожидаемый результат:**
```lua
local client = create_user_api_client("https://api.example.com")
local request = client:get_user(123)
-- {method = "GET", url = "https://api.example.com/users/123", ...}
```

**Время генерации:** ~12 секунд

---

#### 4️⃣ Data Transformer (⭐⭐⭐⭐☆)
**Файл:** `examples/mts_hackathon/04_data_transformer.txt`

**Задача:** Трансформация JSON между форматами разных API

**Что демонстрирует:**
- Парсинг JSON
- Маппинг полей (firstName → first_name)
- Работу с вложенными структурами
- Flatten/nest операции

**Ожидаемый результат:**
```lua
function transform_user_data(api_a_json)
    -- API A format → API B format
    -- Nested objects, field mapping, type conversion
    return api_b_json
end
```

**Время генерации:** ~15 секунд

---

#### 5️⃣ Event Processor (⭐⭐⭐⭐⭐)
**Файл:** `examples/mts_hackathon/05_event_processor.txt`

**Задача:** Обработка событий из Kafka и сохранение в PostgreSQL

**Что демонстрирует:**
- Парсинг Kafka сообщений
- Генерацию SQL запросов
- Batch processing
- Error handling
- **Комбинацию нескольких паттернов** (Kafka + PostgreSQL + JSON)

**Ожидаемый результат:**
```lua
function process_events(kafka_messages)
    -- Parse Kafka → Validate → Build SQL → Batch insert
    return {success_count, error_count, sql_queries}
end
```

**Время генерации:** ~18 секунд

## 🔧 Troubleshooting

### Проблема: Ollama не запускается

**Симптомы:**
```
Error: Failed to connect to Ollama at http://localhost:11434
```

**Решение:**
```bash
# Проверить статус
docker ps | grep ollama

# Перезапустить
docker-compose restart ollama

# Проверить логи
docker logs localscript-ollama
```

---

### Проблема: Модель не загружена

**Симптомы:**
```
Error: model 'qwen2.5-coder:7b' not found
```

**Решение:**
```bash
# Загрузить модель вручную
docker exec localscript-ollama ollama pull qwen2.5-coder:7b

# Проверить список моделей
docker exec localscript-ollama ollama list
```

---

### Проблема: RAG база не инициализирована

**Симптомы:**
```
Warning: ChromaDB collection not found
```

**Решение:**
```bash
# Инициализировать RAG базу
python -m rag.cli init

# Проверить количество документов
python -m rag.cli search "kafka" --top-k 3
```

---

### Проблема: Медленная генерация (>30 сек)

**Причины:**
- CPU inference (нет GPU)
- Недостаточно RAM
- Большая модель

**Решение:**
```bash
# Использовать меньшую модель
docker exec localscript-ollama ollama pull qwen2.5-coder:3b

# Обновить .env
LLM_MODEL=qwen2.5-coder:3b

# Перезапустить
docker-compose restart localscript
```

---

### Проблема: Web UI не открывается

**Симптомы:**
```
Error: Connection refused at http://localhost:8000
```

**Решение:**
```bash
# Проверить статус контейнера
docker ps | grep localscript

# Проверить логи
docker logs localscript-app

# Проверить порты
netstat -an | grep 8000

# Перезапустить
docker-compose restart localscript
```
