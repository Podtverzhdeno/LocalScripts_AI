# Enhanced Execution Graph

Интерактивная визуализация pipeline в стиле LangSmith, но локально и бесплатно.

## Возможности

### 1. Интерактивный граф выполнения

- **Real-time обновления** — граф показывает текущий активный узел
- **Анимированные переходы** — подсвечиваются рёбра при переходе между узлами
- **Цветовая индикация**:
  - 🟢 Зелёный — успешное выполнение
  - 🟡 Жёлтый — retry/улучшение кода
  - 🔴 Красный — ошибка/fail

### 2. Кликабельные узлы

Клик на узел агента автоматически открывает соответствующие файлы:

- **Generator** → последний `iteration_N.lua`
- **Validator** → последний `iteration_N_errors.txt` (если есть ошибки)
- **Reviewer** → `report.md`

### 3. Статистика выполнения

Каждый узел показывает:
- Количество выполнений (badge в правом верхнем углу)
- Время последнего запуска
- Описание агента

### 4. Информационная панель

В правом верхнем углу графа:
- **Iteration** — текущая итерация
- **Status** — текущий статус (Generating code, Validating, etc.)
- **Time** — время выполнения в секундах

### 5. Timeline (временная шкала)

Кнопка "Timeline" открывает модальное окно с:
- Полной историей выполнения по итерациям
- Временными метками каждого события
- Статистикой по агентам (сколько раз каждый запускался)

### 6. Hover tooltips

При наведении на узел агента показывается:
- Название агента
- Описание функции
- Количество выполнений
- Время последнего запуска
- Подсказка "Click to view logs"

## Технические детали

### Архитектура

```
frontend/
├── graph.js          # D3.js визуализация (улучшенная версия)
├── session.html      # Страница сессии с графом
└── app.js            # Общие утилиты
```

### WebSocket события

Backend отправляет события через WebSocket:

```javascript
{
  "event": "node_enter",
  "node": "generate",      // ID узла
  "iteration": 1,
  "status": "generating"
}
```

Frontend обрабатывает их и обновляет граф в реальном времени.

### D3.js граф

Класс `PipelineGraph`:

```javascript
// Инициализация
let graph = new PipelineGraph('graphContainer');

// Установка активного узла
graph.setActiveNode('generate', iteration);

// Сброс
graph.reset();

// Получение истории
const history = graph.getHistory();

// Получение статистики
const stats = graph.getStats();
```

## Использование

1. Запустите сервер:
```bash
python api/server.py
```

2. Откройте http://127.0.0.1:8000

3. Создайте новую задачу

4. Перейдите на страницу сессии — граф будет обновляться в реальном времени

## Кастомизация

### Изменить цвета узлов

В `graph.js`:

```javascript
this.nodes = [
    { id: 'generate', label: 'Generator', color: '#10b981' },
    // ...
];
```

### Добавить новый узел

1. Добавьте в `this.nodes`:
```javascript
{ id: 'my_node', label: 'My Node', x: 300, y: 200, type: 'agent', agent: 'my_agent' }
```

2. Добавьте рёбра в `this.edges`:
```javascript
{ from: 'generate', to: 'my_node', label: 'next' }
```

3. Обновите backend для отправки событий `node_enter` с `node: 'my_node'`

## Troubleshooting

### Граф не обновляется

- Проверьте WebSocket соединение (зелёная точка в "Live Log")
- Откройте DevTools → Console для ошибок
- Убедитесь, что backend отправляет события `node_enter`

### Клик на узел не работает

- Проверьте, что файлы существуют в session директории
- Откройте DevTools → Console для ошибок API запросов

### Tooltip не показывается

- Проверьте, что `node.type === 'agent'`
- Убедитесь, что CSS для `.graph-tooltip` загружен
